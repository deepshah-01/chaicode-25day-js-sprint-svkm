## Lecture 2: JavaScript Setup

### Steps I followed

1. Created one folder with name `JS-Hindi`.
2. Opened that folder in VS Code.
3. Created one file named `test.js`.
4. Wrote my first JavaScript line:

```js
console.log("Kanishka");
```

### What I learned in Lecture 2

- This line is source code which has special meaning.
- It is not like a normal text file where we write anything.
- JavaScript file has `.js` extension.
- We can execute `.js` file, not `.txt` file.
- For many years, JavaScript was mainly used in browsers.
- Like we create `index.html` file and write JavaScript inside it or attach JavaScript file with it.
- But now JavaScript can also work standalone.
- JavaScript is used in backend, mobile development, and many other places.

### JavaScript Runtime Environment

- Node.js is a JavaScript runtime environment.
- Node.js helps us to run JavaScript files outside the browser.
- Deno is also another JavaScript runtime environment.
- Ryan Dahl is the main lead developer/founder of Node.js and Deno.

### Code Editors

Some code editors are:

- VS Code
- JetBrains Fleet
- Sublime Text

VS Code is recommended because it is easy and mostly used by developers.

### How to Run JavaScript File

To check Node.js version:

```bash
node -v
```

To run JavaScript file:

```bash
node test.js
```

Example code:

```js
console.log("Kanishka");
console.log("Day 01 JavaScript Sprint Started");
```

## Lecture 3: GitHub Workspace and Running Code

### What I learned

- In this lecture, I learned how to work on GitHub.
- First, we have to create a repository on GitHub.
- After creating repository, we can open GitHub Workspace / Codespace.
- GitHub Workspace is like VS Code in browser.
- We can create files, write code, open terminal, and run code there.
- It is helpful when local VS Code or system setup is not working properly.
- We can run JavaScript code directly in GitHub Workspace.

### Steps I followed

1. Created one repository on GitHub.
2. Opened that repository in GitHub Workspace / Codespace.
3. Created one JavaScript file named `test.js`.
4. Wrote basic JavaScript code in that file:

```js
console.log("Kanishka");
console.log("Running JavaScript code in GitHub Workspace");
```

5. Opened terminal in GitHub Workspace.
6. Checked Node.js version using:

```bash
node -v
```

7. Ran my JavaScript file using:

```bash
node test.js
```

8. Output was shown in terminal.
9. After running code, I learned how to add, commit, and push code on GitHub.

### Commands Used

```bash
node -v
node test.js
git add .
git commit -m "Day 01: JavaScript Introduction and GitHub Workspace - Kanishka Wani"
git push
```

### Important Points

- GitHub Workspace is a cloud-based coding environment.
- We can run JavaScript code directly in GitHub Workspace.
- To run any JavaScript file, we use:

```bash
node filename.js
```

Example:

```bash
node test.js
```

- After completing code, we have to push our work on GitHub.
- GitHub helps us to store daily progress and show proof of work.
